Working of Backend

Overview
--------
The backend processes an equation entered manually through a linear pipeline:

1. Lexer
2. Parser
3. Solver
4. Constraint Engine
5. Output

Each stage receives the result from the previous stage and transforms it into a form suitable for the next stage.

Flow and Connections
--------------------
- Manual expression string -> `lexer.py` produces tokens.
- Tokens -> `parser.py` builds an equation representation and converts it into coefficients.
- Coefficients and target -> `solver.py` finds integer solutions.
- Solutions -> `constraints.py` filters solutions by any extra rules.
- Final solutions -> output returned by `engine.py`.

Detailed Steps
--------------

1. Lexer (`backend/core/lexer.py`)
   - Role: convert a text expression into tokens.
   - Input: a string like `x + 2*y = 10`
   - Output: a token list such as `[VARIABLE:x, PLUS, NUMBER:2, MUL, VARIABLE:y, EQUAL, NUMBER:10]`

   Important code:
   ```python
   if "0" <= char <= "9":
       start = i
       while i < n and "0" <= expression[i] <= "9":
           i += 1
       append(Token(TokenType.NUMBER, int(expression[start:i])))
   ```
   This reads whole numbers and stores them as `NUMBER` tokens.

   ```python
   token_type = _OPERATOR_MAP.get(char)
   if token_type:
       append(Token(token_type))
       i += 1
   ```
   This converts characters like `+`, `-`, `*`, `/`, `=` and parentheses into operator tokens.

2. Parser (`backend/core/parser.py`)
   - Role: read tokens and build a structured expression tree, then turn it into a solver-friendly equation.
   - Output: a dictionary like `{'coefficients': {'x': 10, 'y': 20}, 'target': 100}`.

   Important code:
   ```python
   left = self.expr()
   if self.current() and self.current().type == TokenType.EQUAL:
       self.eat(TokenType.EQUAL)
       right = self.expr()
       eq = Equation(left, right)
       return self.build_equation(eq)
   ```
   This ensures the parser recognizes the `=` and constructs an equation.

   The parser supports implicit multiplication and operator precedence.
   A key function is `build_equation`, which converts the AST into coefficients:
   ```python
   left_coeffs, left_const = self.eval_poly(eq.left)
   right_coeffs, right_const = self.eval_poly(eq.right)
   coeffs = {}
   for k, v in left_coeffs.items():
       coeffs[k] = coeffs.get(k, 0) + v
   for k, v in right_coeffs.items():
       coeffs[k] = coeffs.get(k, 0) - v
   target = right_const - left_const
   return {'coefficients': coeffs, 'target': target}
   ```
   This moves all variable terms to the left side and constants to the right side.

3. Solver (`backend/core/solver.py`)
   - Role: find non-negative integer solutions for the equation using coefficients.
   - Input: `{'coefficients': {'x': 10, 'y': 20}, 'target': 100}`
   - Output: list of solutions like `[{'x': 2, 'y': 4}]`.

   Important code:
   ```python
   if not is_valid(coeffs, target):
       return []
   coeffs = {k: v for k, v in coeffs.items() if v != 0}
   variables = list(coeffs.keys())
   variables.sort(key=lambda v: -coeffs[v])
   ```
   This removes zero coefficients, sorts variables by coefficient size, and picks the best solving strategy.

   For one variable:
   ```python
   if target % coeff != 0:
       return []
   value = target // coeff
   return [{var: value}]
   ```
   For two variables it loops through one variable and checks if the remainder divides evenly by the other coefficient.

   For many variables it backtracks with pruning:
   ```python
   if remaining < 0:
       break
   current[var] = value
   backtrack(index + 1, current, new_remaining)
   ```
   This avoids exploring impossible values early.

4. Constraint Engine (`backend/core/constraints.py`)
   - Role: apply extra rules like `x >= 1` or `y != 3` after solutions are generated.
   - Input: a solution list plus constraint strings.
   - Output: filtered solutions.

   Important code:
   ```python
   for var, op_func, val in parsed:
       if var not in sol or not op_func(sol[var], val):
           valid = False
           break
   ```
   This checks each solution against every constraint.

   Constraint parsing supports `>`, `<`, `>=`, `<=`, `==`, and `!=`.

5. Output (`backend/core/engine.py`)
   - Role: tie everything together and return the final result summary.
   - Important flow:
   ```python
   tokens = tokenize(expression)
   parsed = Parser(tokens).parse()
   solutions = solve(parsed)
   if constraints:
       solutions = apply_constraints(solutions, constraints)
   return {
       'total_solutions': len(solutions),
       'shown_solutions': len(solutions[:10]),
       'solutions': solutions[:10],
   }
   ```

How to Explain It Simply
------------------------
- `lexer.py` breaks the input string into tokens so the program understands numbers, variables and operators.
- `parser.py` checks the token order, builds a math tree, and converts the equation into coefficients plus a target value.
- `solver.py` finds valid whole-number solutions for the equation using those coefficients.
- `constraints.py` removes solutions that do not meet extra rules.
- `engine.py` runs all these steps and returns the final output.

This is the backend’s working flow, with each file handling one clear stage and passing its result to the next stage.
